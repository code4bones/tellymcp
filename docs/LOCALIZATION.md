# Localization Plan

This file describes the current localization target for TellyMCP.

## Current Rule

Localize human-facing UI only.

Do not localize:

- MCP tool names
- structured payload fields
- `TOOLS.md`
- logs by default

## Human-Facing Surfaces

Should be localized:

- Telegram menus
- Telegram notices
- project/member labels where text is generated by UI
- browser/storage/screenshot menu text
- Live approval text
- webhook/polling operational UI notices when shown to users

Not required yet:

- internal developer docs
- protocol field names
- console/runtime logs

## Active Locale Sources

Current bundled locales:

- `en`
- `ru`

Resolution order:

1. explicit saved user preference
2. Telegram `language_code`
3. fallback `en`

## Naming Rules

Use semantic keys.

Good examples:

- `menu.sessions.title`
- `menu.browser.open`
- `collab.project.members`
- `live.approval.request`
- `storage.empty`

Avoid keys that encode legacy products such as inbox/local-linking unless they still map to real current UI.

## Maintenance Rule

If a user-facing screen is removed or renamed:

- remove the old localization keys
- do not leave dead `inbox`, `pairing`, or `local partner` copy behind

Localization files should reflect the current gateway-first product model.
