# Руководство пользователя: Files

Этот документ описывает работу файлового менеджера простым языком: как открывать каталоги, загружать файлы, настраивать доступ и пользоваться дополнительными возможностями.

## Что такое Files

`Files` — это файловый менеджер внутри приложения.

С его помощью можно:
- просматривать дерево папок;
- открывать содержимое папок;
- загружать файлы;
- создавать, переименовывать и удалять папки;
- открывать и скачивать файлы;
- смотреть предварительный просмотр файлов;
- настраивать доступ к файлам и папкам.

## Как устроен экран

Экран разделен на 3 зоны:

1. Слева сверху — дерево папок  
   Здесь видно структуру каталогов.

2. Справа — содержимое текущей папки  
   Здесь отображаются файлы и вложенные папки.

3. Слева снизу — панель информации  
   Здесь показывается информация о выбранной папке или файле.

Размер областей можно менять мышкой.

## Scope

В верхней части дерева есть список `scope`.

`Scope` — это отдельное пространство файлов.  
Проще говоря, это отдельный корень файловой структуры.

Что можно делать:
- переключаться между scope;
- создавать новый scope.

При создании нового scope система создает для него отдельный корневой каталог `/`.

## Работа с деревом папок

### Открытие папки

- Один клик по папке в дереве:
  - выбирает папку;
  - показывает ее содержимое справа.

- Двойной клик по папке:
  - раскрывает или сворачивает ветку дерева.

### Что видно у папки

Справа в строке папки отображаются:
- значок доступа:
  - `🌐` — public
  - `🔒` — private
- значок локальных правил:
  - `🔐` — на этой папке есть собственные правила доступа
- счетчик папок;
- счетчик файлов.

## Правая панель: содержимое папки

Здесь отображаются:
- вложенные папки;
- файлы.

Для каждого элемента можно:
- выбрать его одним кликом;
- открыть двойным кликом;
- увидеть тип;
- увидеть размер;
- увидеть дату изменения.

### Что означает значок

- `📁` — папка
- `📄` и другие значки — файл
- `🌐` — public
- `🔒` — private
- `🔐` — на узле заданы локальные правила доступа

## Нижняя панель: информация

### Если выбрана папка

Показывается:
- имя папки;
- содержимое папки в компактном виде;
- кнопки управления;
- текущий режим доступа.

### Если выбран файл

Показывается:
- имя файла;
- размер;
- тип;
- ссылка на файл;
- предварительный просмотр, если формат поддерживается;
- текущий режим доступа.

## Что можно делать с файлами и папками

### Создать папку

Кнопка:
- `＋`

Создает новую папку в текущем каталоге.

### Загрузить файл

Кнопка:
- `⤴`

Файл загружается в текущую открытую папку.

Также можно перетащить файл мышкой в правую область.

### Переименовать

Кнопка:
- `✎`

Позволяет изменить имя выбранной папки или файла.

### Удалить

Кнопка:
- `🗑`

Удаляет выбранный элемент.

Если удалить папку, удаляется вся ее вложенная структура.

### Открыть файл

Обычно это делается через:
- двойной клик;
- либо через preview/ссылку в панели информации.

### Скачать файл

Файл скачивается с понятным именем, которое вы видите в файловом менеджере.

## Предварительный просмотр

Для поддерживаемых типов можно сразу увидеть содержимое файла:
- изображения;
- PDF;
- текстовые файлы;
- медиа и некоторые другие форматы.

Если формат нельзя показать внутри страницы, система предложит открыть или скачать файл.

## Права доступа

Для файлов и папок можно настраивать доступ.

Кнопка:
- `🔐`

Она открывает окно настройки доступа.

## Visibility: private / public / inherit

У каждого файла и каталога есть режим видимости.

### `private`

Узел виден авторизованным пользователям системы.

Это основной рабочий режим для внутренних файлов.

### `public`

Узел открыт для чтения и просмотра без обычных ограничений private-зоны.

### `inherit`

Узел наследует режим от родительской папки.

Пример:
- если папка `private`,
- а файл внутри стоит `inherit`,
- значит файл тоже будет `private`.

## Локальные правила доступа

Кроме `public/private`, можно задавать локальные правила.

Они нужны для точечной настройки доступа на отдельных папках и файлах.

Правило состоит из:
- типа:
  - `Role`
  - `Group`
  - `User`
- значения;
- действия:
  - `Allow`
  - `Deny`
- прав:
  - `Read`
  - `List`
  - `Write`
  - `Delete`

### Что означают права

- `Read` — читать и открывать файл/узел
- `List` — видеть содержимое папки
- `Write` — создавать или загружать внутрь
- `Delete` — удалять

### Для чего нужны rules

Если говорить просто:
- `private` определяет, что узел живет во внутренней зоне приложения;
- `rules` позволяют тонко ограничить или разрешить доступ к конкретной папке или файлу.

То есть обычно схема такая:
- вся рабочая структура живет как `private`;
- чувствительные разделы дополнительно ограничиваются правилами.

## Как выбрать пользователя, роль или группу

В правиле можно:
- ввести значение вручную;
- либо нажать кнопку поиска внутри поля и выбрать из списка.

В отдельном окне выбора доступны:
- поиск;
- переключение страниц;
- выбор готового значения.

## Как работает наследование правил

Права наследуются сверху вниз.

Это значит:
- если на папке заданы правила,
- дочерние узлы могут их унаследовать.

Если на файле или папке задать собственные локальные правила, они будут применяться к самому этому узлу.

## ACL Index

Кнопка:
- `🧭 ACL`

Открывает отдельное окно, где можно быстро увидеть:
- какие узлы имеют собственные правила;
- какие правила и к каким узлам привязаны.

### Режим 1: `Nodes with rules`

Показывает список папок и файлов, на которых есть локальные правила.

Для каждого узла можно:
- перейти к нему;
- быстро очистить локальные правила кнопкой `Clear`.

### Режим 2: `Rules index`

Показывает правила в сгруппированном виде:
- кому выдано правило;
- какое это правило;
- к каким узлам оно привязано.

Из этого режима тоже можно:
- перейти к узлу;
- убрать локальные правила через `Clear`.

### Что делает `Clear`

`Clear` удаляет локальные правила только у выбранного узла.

Важно:
- `Visibility` при этом не меняется;
- унаследованные правила от родительской папки могут продолжать действовать.

## Интеграция с Tools

В панели информации есть кнопка:
- `🔎`

Она открывает дополнительные технические инспекторы:
- для папки — VFS Inspector;
- для файла — VFS Inspector и MinIO Inspector.

Обычно обычному пользователю это не нужно, но для администраторов и разработчиков это удобно.

## Что запоминается

Система запоминает:
- выбранный scope;
- текущую открытую папку;
- выбранный элемент;
- положение разделителей экрана.

Если вы уйдете на другую вкладку и вернетесь, система постарается восстановить предыдущее состояние.

## Полезные сценарии

### Сделать папку внутренней, но доступной только части пользователей

1. Выберите папку.
2. Нажмите `🔐`.
3. В `Visibility` оставьте `private`.
4. Добавьте нужные правила для ролей, групп или пользователей.
5. Сохраните.

### Сделать файл открытым

1. Выберите файл.
2. Нажмите `🔐`.
3. В `Visibility` выберите `public`.
4. Сохраните.

### Ограничить доступ только для одной роли

1. Откройте `🔐`.
2. Добавьте правило.
3. Выберите `Role`.
4. Найдите нужную роль.
5. Отметьте нужные права.
6. Выберите `Allow`.
7. Сохраните.

### Найти, где вообще стоят правила

1. Нажмите `🧭 ACL`.
2. Откройте:
   - `Nodes with rules`, если хотите найти конкретные папки/файлы;
   - `Rules index`, если хотите посмотреть правила по ролям/группам/пользователям.

## Если что-то выглядит странно

Если файл или папка:
- не открывается;
- пропала из дерева;
- не скачивается;
- показывает неожиданный доступ,

проверьте:
- `Visibility`
- локальные rules
- `ACL Index`
- наследование от родительской папки

## Кратко

Если совсем коротко:
- дерево слева — навигация по папкам;
- список справа — содержимое папки;
- нижняя панель — информация и preview;
- `🔐` — настройка доступа;
- `🧭 ACL` — обзор всех локальных правил;
- `🌐` — public;
- `🔒` — private;
- `🔐` на узле — есть локальные правила.
